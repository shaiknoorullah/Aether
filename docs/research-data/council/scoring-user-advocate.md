# User Advocate — RICE Scoring

> **Perspective**: I champion user needs over technical elegance. Every score asks: does a real person benefit from this, can they discover it, and does it simplify or complicate their day?
>
> **Scoring biases applied**:
> - Overweight features with broad reach (all users > niche segments)
> - Automatic/invisible features score higher than manual ones
> - Penalize features that add visible complexity without clear user benefit
> - First impressions and "wow" moments weighted heavily
> - Accessibility is non-negotiable, not a nice-to-have

## Rationale for Top 10

**1. Native Ad/Tracker Blocking (RICE 12.50)** — 900M+ ad-block users worldwide. This is the single feature most correlated with browser adoption (Brave's entire growth story). Every user benefits immediately, zero learning curve, instant "wow" moment on first page load. Maximum reach, maximum impact, high confidence, moderate effort.

**2. Notification Spam Blocking (RICE 12.15)** — 300% increase in malicious push notifications. Users unknowingly grant permissions and can't revoke them. This is invisible protection — the best kind of feature. Extremely cheap to implement (effort 2), protects nearly everyone, and removes a daily annoyance without users needing to do anything.

**3. Vertical Tab Sidebar (RICE 10.80)** — Chrome took 17 years. Users with 10+ tabs can't read horizontal titles. Vertical tabs are self-evidently better — readable titles, better use of widescreen real estate. Low effort, high reach. This is a "why didn't my old browser do this?" moment.

**4. AI Page Summarization (RICE 9.60)** — The #1 use case across all AI extensions. One-click summary is the most discoverable AI feature. Zero learning curve, immediate value, works for articles, PDFs, and YouTube. This is the "show, don't tell" moment for AI in a browser.

**5. WebExtension API Support (RICE 8.33)** — The #1 reason users leave keyboard-first browsers. Without uBlock Origin and password managers, the browser is DOA. High effort but there's no workaround — you either have it or you don't. Users don't think about this as a "feature"; its absence is a dealbreaker.

**6. Frictionless Migration from Chrome (RICE 8.10)** — 34% of non-switchers stay due to inertia. The migration experience IS the first impression. If importing bookmarks, passwords, and history takes more than one click, users bounce. This is the gateway through which every user enters.

**7. Tab Graveyard / Recovery (RICE 7.20)** — Tab hoarding is driven by fear of losing something. A trivially searchable, state-preserving recovery system eliminates that fear. This is invisible infrastructure that changes behavior — users close tabs freely knowing recovery is effortless. Low effort, broad reach.

**8. Memory-Efficient Tab Management (RICE 7.20)** — Qutebrowser users report 100GB RAM with a few dozen tabs. Users don't want to think about memory. If the browser is sluggish, nothing else matters. This is table stakes competence that affects literally every user session.

**9. LLM Sidebar Chat (RICE 5.76)** — The most natural AI interaction pattern for browsing. Context-aware chat on the current page is discoverable, useful, and matches existing mental models (sidebar panels). Not a Must because the browser functions without it, but it's the primary AI interaction surface.

**10. Tab Unloading/Hibernation (RICE 5.40)** — Edge dropped from 3GB to 500MB with tab suspension. Users don't notice it working — they just notice their laptop isn't overheating. Invisible, automatic, benefits everyone who keeps more than 10 tabs open (which is most users).

## Rationale for "Must Have" Picks

Nine features are non-negotiable:

1. **Native Ad/Tracker Blocking** — Without this, we're asking users to install an extension on day one, which means we've already failed the "just works" test. Brave proved this drives adoption at scale. MV3 is killing extension-based blocking; native is the future.

2. **WebExtension API Support** — Users have invested in extensions (password managers, specific workflow tools). Asking them to abandon that ecosystem is asking them not to switch. This is the difference between "I'll try it" and "I can't use this."

3. **Vertical Tab Sidebar** — In 2026, horizontal tabs on a widescreen monitor is indefensible. This is a baseline expectation for any new browser targeting knowledge workers. Users who see it won't go back.

4. **Memory-Efficient Tab Management** — A browser that chews through RAM is unshippable. Users with 8GB laptops (still the majority globally) will hit walls immediately. This isn't a feature; it's engineering competence.

5. **Efficient Rendering Engine** — If banking sites, video, and SPAs break, nothing else matters. The browser's primary job is rendering the web correctly. Users will not tolerate "works on most sites."

6. **Cross-Platform Full Parity** — Dia and SigmaOS are macOS-only, which caps their market. Launching without Windows or Linux support excludes the majority. Mobile can follow, but desktop parity is non-negotiable.

7. **Frictionless Migration from Chrome** — 65-71% of users are on Chrome. Every new Aether user is a Chrome refugee. If they can't bring their data, they won't come. One-click import is the entry door.

8. **Keyboard-Only Full Navigation** — This is an accessibility requirement, not a power-user feature. Every surface of the browser must be keyboard-navigable. This is legally relevant (WCAG), morally required, and also serves the RSI/motor-impairment population. A browser that can only be used with a mouse fails a significant user population.

9. **Notification Spam Blocking** — Malicious notification abuse is epidemic. Users grant permissions by accident and get bombarded. Blocking by default is trivial to implement (effort 2) and prevents a class of harm that disproportionately affects less technical users. This is protection, not a feature.

## Controversial Picks

### Project-First Workspaces ranked #12, not top 5
Despite having the highest evidence count (10 reports), I scored this as Should, not Must. Reason: workspaces are powerful but add conceptual overhead for new users. A user in minute one doesn't have "projects" — they have tabs. The workspace model needs to be introduced gradually, not forced. Arc tried to make Spaces mandatory and alienated casual users. The right approach is making workspaces discoverable after the user accumulates enough tabs to feel pain, not as a first-run paradigm.

### Local/On-Device AI ranked #37
The engineering and privacy communities love this. Users don't care where inference runs — they care that it works. A user will never say "I wish this summary ran on my GPU." They will say "this summary is slow" (because on-device models are slower than cloud). Local AI is important for our privacy story and positioning, but its RICE score reflects that the direct user experience benefit is often negative (slower, worse quality) until hardware catches up. It's a Should because of brand positioning, not user demand.

### Agent Automation ranked #45 (Won't)
Agent reliability is 19.7% for 10-step workflows. Shipping this at launch means shipping a feature that fails 80% of the time. From a user advocate perspective, this is worse than not having it — it teaches users that AI features are unreliable. The reputational cost of a broken agent is higher than the opportunity cost of waiting. This belongs in v2 when reliability reaches >90%.

### Native Vim Keybindings ranked #32 (Could)
This will anger the core community. Vim users are vocal and the research shows 660K+ keyboard-first browser users. But from a pure user-reach perspective, this is <1% of the target market. I scored it as Could — not because it's unimportant for differentiation, but because my role is to represent ALL users, not the loudest segment. The Security Architect or CTO should fight for this; I fight for the 99%.

### Notification Spam Blocking ranked #2 (Must)
Other council members may view this as a minor quality-of-life feature. I ranked it #2 because it has the highest RICE efficiency (effort 2 for reach 9). The ratio of user benefit to engineering cost is unmatched. It's also a "protection" feature — it prevents harm to the least technical users, which is exactly who an AI-native browser needs to reach beyond the early adopter segment. Every notification permission popup a user never sees is trust earned.

### AI Page Summarization ranked above Project-First Workspaces
Page summarization is the "show, don't tell" moment for AI in the browser. A user visits an article, clicks one button, gets a summary. Instant value demonstration with zero learning curve. Workspaces require understanding a new mental model. In terms of first-impression impact, summarization wins despite workspaces being the strategically deeper feature. Different time horizons: summarization wins day one, workspaces win month one.

### Cross-Device Sync ranked #15 (Should, not Must)
This will surprise some. Sync is expected, but it's not expected at launch for a new browser. Users tolerate "sync coming soon" — they don't tolerate "extensions don't work." The effort (7) is high for a launch feature when there are cheaper wins available. Privacy-preserving sync is particularly hard to get right. Ship it in v1.1, not v1.0.

### Theming ranked #18 (Should)
92% want personalization, but theming is a retention feature, not an acquisition feature. Nobody switches browsers for themes. They stay for themes. This is a post-launch investment in stickiness, appropriately ranked as Should.
