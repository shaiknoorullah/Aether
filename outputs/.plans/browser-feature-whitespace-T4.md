# T4: Neurodivergent Accessibility Features + Whitespace (2026)

## Objective
Map accessibility features relevant to neurodivergent users (ADHD, autism, dyslexia, sensory processing) and identify what NO browser provides.

## Scope
Research browsers: Firefox, Chrome, Edge, Brave, Vivaldi, Arc, Safari, Zen, Orion. Also cover: accessibility extensions (Helperbird, BeeLine Reader, Midnight Lizard, StayFocusd, LeechBlock, Bionic Reading, OpenDyslexic), OS-level accessibility features, and academic/clinical research on neurodivergent web browsing needs.

## Key Questions
1. What neurodivergent-relevant features do browsers ship natively?
2. What exists only as extensions?
3. What does NO browser or extension provide? Look for:
   - Cognitive load indicators (visual signal when page complexity is high)
   - Focus scaffolding (progressive disclosure of page content, guided reading mode)
   - Stimuli control beyond dark mode (animation freezing, motion reduction, color temperature per-site, sound management)
   - ADHD-specific tab management (auto-archive, gentle nudges, time-on-site awareness)
   - Reading ruler / line guide built into rendering
   - Customizable information density (not just zoom — restructure layout)
   - Task-anchored browsing (tie browser state to a task, minimize distractions)
   - Sensory profiles (save and switch complete sensory preference sets)
   - Executive function support (browser-native timers, break reminders, session goals)
   - Predictable UI (no surprise layout shifts, consistent controls)
   - Dyslexia-optimized rendering engine (syllable highlighting, phonetic aids, not just font swap)
   - Autism-friendly defaults (reduced social/notification pressure, clear navigation)
   - Hyperlink previews with context (reduce anxiety about where links go)
4. What does clinical/academic research say about neurodivergent web browsing needs that browser makers ignore?

## Output
Write to `outputs/.drafts/browser-feature-whitespace-research-t4.md` with:
- Feature matrix: browser × ND-relevant feature
- Whitespace features with evidence
- Source URLs including academic sources where available
- Notes on why gaps exist (awareness, market size, medical-device concerns)

## Rules
- Use web search for current docs, research summaries, extension listings
- Do NOT fetch PDF files — cite paper titles and metadata only
- Every claim needs a source URL
- Mark uncertain items as "unverified"
- Be specific about which neurodivergent conditions each feature serves
