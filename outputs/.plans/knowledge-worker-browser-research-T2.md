# Researcher Brief T2 — Research-and-Capture Workflows & Note-Taking Integration

## Your job
Search for and collect evidence on how knowledge workers and researchers capture, annotate, and resurface web content, and how browsers integrate (or fail to integrate) with note-taking tools.

1. **Research-and-capture workflows**: How do researchers clip/save/annotate web content? Tools used: Zotero, Hypothesis, Readwise, Pocket, Instapaper, browser bookmarks, browser extensions. Adoption/usage stats where available. Friction points at the "read → save → resurface" loop.

2. **Browser bookmark behavior**: How many bookmarks go unused? Research on bookmark decay, "bookmark and forget." Any user studies on bookmark retrieval rates.

3. **Note-taking browser integration**: Which note-taking tools have browser extensions (Notion, Obsidian, Roam, Logseq, Evernote, Notion Web Clipper, etc.)? Usage friction data. Survey data on which tools researchers use alongside the browser.

4. **Reading/annotation tools**: Hypothesis (web annotation), Readwise Reader, Matter, Instapaper. User counts, growth, friction studies.

5. **Tab-to-note friction**: How do users bridge "I have 47 open tabs because I'm scared to lose them" to "organized notes"? Any UX research or survey data on this behavior pattern.

## Search angles
- "web research capture workflow knowledge workers" survey
- "browser bookmarks usage rate forgotten" study
- "Zotero Readwise Hypothesis adoption researchers" statistics
- "note-taking browser extension workflow" user research
- "tab hoarding behavior browser" psychology
